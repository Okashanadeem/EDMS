# EDMS Visual API Book — Phase 1 MVP

This document provides visual request and response examples for the EDMS API (v1). All routes are prefixed with `/api/v1`.

---

## 1. Authentication

### POST `/auth/login`
Authenticates a user and returns a JWT token.

**Request:**
```json
{
  "email": "superadmin@edms.local",
  "password": "Admin@1234"
}
```

**Response (200 OK):**
```json
{
  "success": true,
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "name": "System Super Admin",
      "email": "superadmin@edms.local",
      "role": "super_admin",
      "department_id": null
    }
  }
}
```

---

### PATCH `/auth/change-password`
Allows an authenticated user to change their own password.

**Request:**
```json
{
  "currentPassword": "OldPassword@123",
  "newPassword": "NewPassword@456"
}
```

**Response (200 OK):**
```json
{
  "success": true,
  "message": "Password updated successfully."
}
```

## 2. Departments (Super Admin Only)

### POST `/departments`
Creates a new department.

**Request:**
```json
{
  "name": "Crime Investigation Department",
  "code": "CID"
}
```

**Response (201 Created):**
```json
{
  "success": true,
  "data": {
    "id": 3,
    "name": "Crime Investigation Department",
    "code": "CID",
    "is_active": true
  }
}
```

## 3. Users (Super Admin Only)

### POST `/users`
Creates a new worker account and returns a one-time temporary password.

**Request:**
```json
{
  "name": "Priya Sharma",
  "email": "priya@edms.local",
  "role": "worker",
  "department_id": 3
}
```

**Response (201 Created):**
```json
{
  "success": true,
  "data": {
    "user": {
      "id": 12,
      "name": "Priya Sharma",
      "email": "priya@edms.local",
      "role": "worker",
      "department_id": 3
    },
    "temporaryPassword": "Xk9#mR2pLq"
  }
}
```

## 4. Documents (Worker Only)

### POST `/documents`
Creates and dispatches a document.

**Request (Multipart/Form-Data):**
- `subject`: "Request for Staff"
- `body`: "Detailed body content..."
- `receiver_department_id`: 7
- `file`: (Optional file upload)

**Response (201 Created):**
```json
{
  "success": true,
  "data": {
    "id": 101,
    "outward_number": "CID/OUT/2025/0042",
    "status": "in_transit"
  }
}
```

---

### POST `/documents/:id/pickup`
Claims a document from the department inbox.

**Response (200 OK):**
```json
{
  "success": true,
  "data": {
    "id": 101,
    "inward_number": "REC/IN/2025/0011",
    "status": "picked_up",
    "assigned_to": 12
  }
}
```

**Response (409 Conflict):**
```json
{
  "success": false,
  "error": "This document has already been picked up by another worker."
}
```

## 5. Internal Testing Utilities

### POST `/test/cleanup`
Clears all dynamic data (documents, logs, non-admin users).

**Request Headers:**
- `x-internal-test: true` (or `NODE_ENV=development`)

**Response (200 OK):**
```json
{
  "success": true,
  "message": "Cleanup successful. All dynamic data removed."
}
```
