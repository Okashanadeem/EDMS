# EDMS Visual API Book — Phase 1.1

This document provides visual request and response examples for the new Phase 1.1 features. All routes are prefixed with `/api/v1`.

---

## 1. Drafts Module

### GET `/drafts`
Lists drafts available to the user. Officers see assistant submissions.

**Response (200 OK):**
```json
{
  "success": true,
  "data": [
    {
      "id": 201,
      "subject": "Budget Proposal FY26",
      "status": "draft",
      "creator_name": "Ravi Sharma",
      "draft_submitted_at": "2026-03-31T14:00:00Z",
      "behalf_of_officer_id": 12,
      "updated_at": "2026-03-31T14:05:00Z"
    }
  ]
}
```

---

### POST `/drafts/:id/submit`
Assistant submits a draft to their assigned Officer.

**Response (200 OK):**
```json
{
  "success": true,
  "message": "Draft submitted to officer for review.",
  "data": { "id": 201, "draft_submitted_at": "..." }
}
```

---

### POST `/drafts/:id/approve`
Officer approves a submitted draft, triggering dispatch.

**Response (200 OK):**
```json
{
  "success": true,
  "message": "Draft approved and dispatched.",
  "outward_number": "DIG/OUT/2026/0004",
  "document_id": 201
}
```

## 2. OTP Delegated Sending

### POST `/otp/request`
Assistant requests an OTP code to be sent to the Officer's email.

**Request:**
```json
{ "document_id": 201 }
```

**Response (200 OK):**
```json
{
  "success": true,
  "message": "OTP sent to officer email."
}
```

---

### POST `/otp/verify`
Assistant provides the OTP shared by the Officer to dispatch the document.

**Request:**
```json
{ "document_id": 201, "otp": "847291" }
```

**Response (200 OK):**
```json
{
  "success": true,
  "message": "OTP verified. Document dispatched on behalf of Officer.",
  "data": {
    "id": 201,
    "outward_number": "DIG/OUT/2026/0004",
    "behalf_otp_used": true
  }
}
```

## 3. Advanced Document Features

### POST `/documents` (Rich Text + Multi-Destination)
Creates a document with CC/BCC and Reference links.

**Request (Multipart/Form-Data):**
- `subject`: "Security Clearance Protocol"
- `body_html`: "<h1>Protocol</h1><p>Update needed...</p>"
- `receiver_department_id`: 5
- `cc[]`: 2
- `cc[]`: 7
- `bcc[]`: 4
- `references[]`: 101
- `is_restricted`: true
- `restricted_to_user_id`: 9

---

## 4. Restriction Behavior

### GET `/documents/:id` (Restricted)
Unauthorized user attempts to access a restricted document.

**Response (200 OK):**
```json
{
  "success": true,
  "data": {
    "id": 305,
    "subject": "Confidential Report",
    "is_restricted": true,
    "is_redacted": true,
    "body_html": null,
    "file_path": null,
    "sender_department_name": "Intelligence"
  }
}
```
